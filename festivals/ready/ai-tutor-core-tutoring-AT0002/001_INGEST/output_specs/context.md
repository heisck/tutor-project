# Context

## Prior Foundation

AT0001 already established the base system:
- Authentication
- Upload flow
- Session base
- Database foundation
- Core infrastructure

AT0002 extends that base into the tutoring engine.

## System Identity

This product is:
- An adaptive tutor
- A guided teaching system
- A mastery-aware learning flow

This product is not:
- A plain summarizer
- A generic chatbot
- An unrestricted answer engine

## Main Components in Scope

- Ingestion Engine
- Atomic Teachable Unit Mapper
- Analysis Engine
- Teaching Planner
- Tutor Engine
- Confusion Signal Engine
- Explanation Diversity Memory
- Assistant Engine
- Session Handoff Engine
- Coverage Audit Engine

## Teaching Model

The system should:
- Teach concept by concept
- Adapt to learner responses
- Detect confusion early
- Require evidence for mastery
- Resume learning cleanly across sessions

## Grounding Model

All teaching and assistant responses must be grounded in uploaded document content and scoped to the correct user and document.
